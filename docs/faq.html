第六部分：常见问题解答 (FAQ)

安装相关问题

Q: 安装时出现权限错误怎么办？

A: 尝试修复 npm 的全局包权限，避免直接使用 sudo。可以搜索 "npm fix permissions" 查找解决方案。

terminal
# 修复 npm 权限的常见方法
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

Q: 安装后命令未找到怎么办？

A: 检查 npm 的全局包路径是否已添加到系统的 PATH 环境变量中。

terminal
# 检查 npm 全局包路径
npm config get prefix

# 检查 PATH 环境变量
echo $PATH

使用相关问题

Q: Gemini 似乎没有理解我的项目文件？

A: 确认您使用了 @ 命令正确注入了文件或目录。使用 /context show 检查当前上下文是否包含了您期望的文件内容。

terminal
# 正确使用 @ 命令注入文件
@gemini.md

# 检查上下文
/context show

Q: 如何重置认证信息？

A: 运行 /auth 命令，里面通常会提供重置或重新登录的选项。

terminal
/auth

Q: 如何查看 Gemini-CLI 的版本信息？

A: 使用 /about 命令可以查看版本号和相关链接。

terminal
/about

故障排除

Q: 命令执行失败怎么办？

A: 首先检查错误信息，然后尝试以下步骤：

  1. 使用 /log 查看详细日志
  2. 确保您的网络连接正常
  3. 检查配置文件是否有语法错误
  4. 尝试重启 Gemini-CLI
terminal
/log

A: 使用 /bug 命令可以引导您提交 Bug 报告。

terminal
/bug

Q: 如何提供使用反馈？

A: 使用 /feedback 命令可以快速提交使用体验反馈。

terminal
/feedback

高级问题

Q: 如何自定义 Gemini-CLI 的提示词？

A: 您可以在 GEMINI.md 文件中添加自定义提示词，然后使用 /memory refresh 命令使其生效。

Q: 如何集成到我的开发环境中？

A: Gemini-CLI 支持与多种开发工具集成：

  • 使用 /mcp 命令配置与 MCP 服务器的连接
  • 使用 /ide 命令启用 IDE 集成
  • 使用 /setup-github 配置 GitHub Actions 工作流

Q: 如何优化性能？

A: 以下是一些优化性能的建议：

  • 定期使用 /compress 压缩长对话历史
  • 合理管理上下文，避免注入不必要的大文件
  • 调整 settings.json 中的 temperature 参数以平衡创造性和性能

斜杠命令速查

会话与历史管理

  • /chat save - 保存当前完整的对话状态
  • /chat resume - 恢复一个之前保存的对话
  • /chat list - 列出所有已保存的会话标签
  • /chat clear - 清空当前对话的历史记录

上下文与记忆管理

  • /memory show - 显示当前所有已加载的 GEMINI.md 文件内容
  • /memory refresh - 重新加载所有记忆文件
  • /memory add "" - 临时向当前会话的记忆中添加文本指令

工具与集成

  • /tools - 列出当前会话所有可用的工具及其状态
  • /extensions - 列出当前活动的扩展
  • /init - 分析当前项目并创建一个量身定制的 GEMINI.md 文件
