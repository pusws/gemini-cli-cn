第四部分：高级技巧与工作流

1. 代码调试工作流

Gemini-CLI 可以成为您代码调试的强大助手，通过结合多种交互模式形成高效的工作流：

terminal
# 1. 使用 @ 命令载入问题代码
@/path/to/file.py

# 2. 提问分析代码问题
分析这段代码可能出现的 IndexError 异常，并给出修复建议。

# 3. 使用 ! 命令运行相关测试
!pytest tests/

# 4. 让 Gemini 直接生成修复后的代码
请根据你的分析，生成修复后的代码。

# 5. 使用 /copy 粘贴修复后的代码
/copy

通过这种方式，您可以快速定位问题、获得修复建议并直接应用修复。

2. 自动化文档生成

Gemini-CLI 可以帮助您自动生成项目文档，节省大量时间：

terminal
# 1. 在项目根目录创建 GEMINI.md，写入指令
echo '所有文档需为 Markdown 格式，包含代码示例。' > GEMINI.md

# 2. 运行 /memory refresh 加载指令
/memory refresh

# 3. 提问生成 API 文档
请为 @./src/ 目录下的所有模块生成 API 文档。

Gemini-CLI 会根据您的代码和指令自动生成结构化的文档。

3. 高级上下文管理

有效管理上下文可以帮助 Gemini-CLI 更好地理解您的需求：

使用 /memory 管理长期记忆

通过 GEMINI.md 文件，您可以为 Gemini-CLI 提供项目的长期记忆：

terminal
# 在 GEMINI.md 中定义项目规范
项目编码规范:
- 使用 4 个空格缩进
- 函数名使用 snake_case
- 类名使用 PascalCase
- 所有函数都需要文档字符串

然后使用 /memory refresh 命令让 Gemini-CLI 加载这些规范。

使用 /context 调试上下文

当 Gemini-CLI 的回答不符合预期时，可以使用 /context show 查看实际发送给模型的上下文：

terminal
/context show

这可以帮助您理解为什么模型会给出特定的回答，并调整您的提示或上下文。

4. 会话管理技巧

合理管理会话可以提高工作效率：

保存和恢复会话

对于复杂的任务，您可以保存当前会话并在稍后恢复：

terminal
# 保存当前会话
/chat save complex-task

# 退出并稍后恢复
/chat resume complex-task

使用 /compress 优化长会话

当会话变得很长时，可以使用 /compress 命令压缩历史记录：

terminal
/compress

这会将长对话历史压缩成简洁的摘要，节省 Token 并提高响应速度。

5. 工具集成技巧

充分利用 Gemini-CLI 的工具集可以极大提升工作效率：

使用 /tools 查看可用工具

了解当前会话中可用的工具及其状态：

terminal
/tools

使用 /yolo 模式加速工作流

在安全的环境中，可以启用 /yolo 模式来自动批准工具执行：

terminal
/yolo

警告：仅在您完全信任当前任务和环境时使用 /yolo 模式。

斜杠命令速查

会话与历史管理

  • /chat save - 保存当前完整的对话状态
  • /chat resume - 恢复一个之前保存的对话
  • /chat list - 列出所有已保存的会话标签
  • /chat clear - 清空当前对话的历史记录

上下文与记忆管理

  • /memory show - 显示当前所有已加载的 GEMINI.md 文件内容
  • /memory refresh - 重新加载所有记忆文件
  • /memory add "" - 临时向当前会话的记忆中添加文本指令

工具与集成

  • /tools - 列出当前会话所有可用的工具及其状态
  • /extensions - 列出当前活动的扩展
  • /init - 分析当前项目并创建一个量身定制的 GEMINI.md 文件
